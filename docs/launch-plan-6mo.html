The opportunity & what's new

AutoMate is a two-sided auto-care marketplace: car owners get AI repair estimates plus real dealer quotes; dealerships compete for the work. The consumer app is a working prototype today (68 screens, offline demo mode, backend + AI estimate/OCR + points + insurance/actuarial modules, real auth/maps/camera UI) — but it is not yet deployed, and the dealer side isn't a product.

This plan adds the missing half — a Dealer SaaS Portal: the web dashboard where dealerships sign in, answer quote requests, manage bookings, set prices/hours, get paid, and see analytics. It's built on the backend that already exists (dealers, quotes, bookings, services, payments are already modeled), so it's a web frontend + a few endpoints — not a second platform — and it directly fixes the biggest risk: a marketplace with no real supply side.

Budget headroom can fund a ~$3–6k contractor sprint to parallelize.

Dealer portal — MVP scope

✅ In: invite-based dealer login · view damage photos + AI estimate → submit quote (price / turnaround / note) · bookings calendar + mark complete · business profile (hours, address, service menu, bundle deals) · Stripe Connect payout onboarding · basic analytics (quotes sent/won, revenue, response time).

🚫 Deferred (post-launch): in-app chat, multi-location, staff roles/permissions, self-serve marketing tools, automated payout reconciliation.

Deliberate lean tradeoffs

• Defer the trained damage-AI model → ship an honest "estimate range" (mock/heuristic engine, disclosed confidence); train post-raise.
• Defer the paid insurance aggregator (Axle) → manual entry + card-scan only.
• Dealer portal = MVP + concierge fallback, not full self-serve on day one.
• Dev partner is equity, not cash — the budget depends on it.
